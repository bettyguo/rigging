# Post-v0 code audit — 2026-05-15

> Findings from the first audit pass after Phase 4. Mix of correctness
> bugs, spec-implementation gaps, and ergonomics. Each item names the
> file and a fix plan. Issues are tracked here and closed by commits
> until the list is empty.

## A. Correctness bugs

### A1. Verifier output signature never verified
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`
**Severity:** security
**Symptom:** ``_run_verifier`` reads the verdict from
`verdict_result.output` and trusts it without re-verifying the
verifier's signature over that output. Any code path that forges a
result could bypass verification.
**Fix:** verify the verifier's JWS envelope (`verdict_result.signature`)
against the verifier's registered public key before accepting the
verdict.

### A2. Contract expiry not enforced at execute time
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`
**Severity:** correctness
**Symptom:** the contract model validates `expires > issued`, but the
runtime never checks `now < contract.expires` before executing.  A
long-queued contract could execute past its `expires`. Per
`rig-contract-v0.md` §6, this should void with `reason_code=expired`.
**Fix:** check expiry at execute boundary, void if past.

### A3. Per-execute timeout missing
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`
**Severity:** correctness (liveness)
**Symptom:** a stuck adapter blocks the rig forever. The contract's
`expires` is the natural timeout but is not wired to a cancel scope.
**Fix:** wrap `Agent.execute` in `anyio.move_on_after(seconds_until_expiry)`;
on timeout, void the contract with `callee_unreachable`.

### A4. `rig.contract.void` spans never emitted
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`
**Severity:** spec conformance
**Symptom:** spec defines `rig.contract.void` for budget overrun,
callee/verifier unreachable, expiry; runtime currently emits
`rig.contract.reject` for these.
**Fix:** distinguish reject (callee declined) from void (runtime
voided) when stamping the terminal span.

### A5. Span hierarchy is flat
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`
**Severity:** trace correctness
**Symptom:** `_open_parent_builder` returns `None` unconditionally;
all spans hang off the recorder root. This means sub-contract spans
appear as siblings of their parent's `rig.execute` instead of children.
**Fix:** thread an explicit `SpanBuilder` argument through `call`,
`_execute_with_span`, and `_run_verifier`; the verifier sub-contract's
spans become children of the parent execute span.

### A6. Example 04 doesn't exercise hierarchical budgets
**File:** `examples/04_cost_attribution/run.py`
**Severity:** demo correctness
**Symptom:** the example calls `rig.call(parent_contract=None)` for
the C step and admits the limitation in a comment. Misses the point.
**Fix:** rewire so B is the caller of C *and* the parent contract is
the A→B contract.

## B. Spec gaps not yet implemented

### B1. `rig.consumed` event emission API absent
**File:** `packages/rigging-trace/src/rigging/trace/recorder.py`
**Symptom:** `trace-v0.md` §3.4 mandates a `rig.consumed` event for
fan-out pruning. `SpanRecord.consumed_contract_ids` is a typed field
but no helper writes it via the recorder.
**Fix:** add `SpanBuilder.set_consumed()` (already exists) — but expose
it via `Rig.set_consumed_for_current(...)` so adapters can mark which
sub-contracts they used as load-bearing.

### B2. Replay rejection has no test
**Symptom:** the contract format relies on `contract_id` uniqueness
plus `expires` for replay rejection. No test asserts this end-to-end.
**Fix:** add a property test that re-submits a contract id and
expects rejection at issuance.

### B3. Trace export / inspect round-trip incomplete
**File:** `packages/rigging-runtime/src/rigging/runtime/rig.py`,
`packages/rigging-trace/src/rigging/trace/inspect.py`
**Symptom:** `rig trace inspect` reads from disk but nothing writes a
trace there. The asciinema demo cannot show inspection.
**Fix:** add `Rig.export_trace(path)` and document a `--trace` flag
on the example scripts.

## C. Type-checking holes

### C1. Missing `py.typed` markers
**Files:** every package under `packages/*/src/rigging/<package>/`.
**Symptom:** `mypy --strict` reports ~80 false-positive
`import-untyped` errors on internal imports.
**Fix:** add `py.typed` files.

### C2. Real mypy `no-any-return` errors
**Files:** `litellm_adapter.py:79,131,136,137`,
`identity/cli.py:40`.
**Symptom:** `Returning Any from function declared to return ...`.
**Fix:** narrow the return paths with explicit `bool(...)` /
`Decimal(...)` / `str(...).encode(...)`.

### C3. Stale `type: ignore` comments
**Files:** `cli.py:58,76`, `rig.py:477`.
**Fix:** remove.

## D. Enhancement opportunities (kept narrow on purpose)

These are the ones worth doing now; everything else goes to
`docs/roadmap.md` instead.

- **D1.** Sync-or-async handler support in `LocalPythonAdapter`.
- **D2.** Centralised policy via a `RigConfig` (max-lifetime,
  recursion cap, default contract budget). Today the constants are
  hard-coded in `rig.py`.
- **D3.** A *vote-ensemble verifier* helper — a thin agent that
  fans a verify call out to N child verifiers and reports the majority
  verdict. Demonstrates composition without a runtime-side feature.
- **D4.** Mid-chain blame attribution. Currently the extractor names
  the leaf agent. When the *issuer* (planner) routed wrongly, the
  leaf isn't actually to blame. Heuristic: if all sub-contracts
  succeeded but the parent's verifier rejects the parent's output,
  the parent is blamed.
- **D5.** A new Example 05 — vote ensemble — to exercise D3.

## Disposition

Order of work for this audit cycle:

1. **C1, C2, C3** — clean mypy first; cheap and prevents future
   regressions from hiding under noise.
2. **A1, A2, A3, A4, A5** — correctness/spec bugs in the runtime,
   in that order.
3. **B1, B3** — close the trace-side gaps; B2 lands as E3.
4. **D1–D5** — feature work that delivers tangible new capability.
5. **E1–E5** — new tests; some land alongside the corresponding fix.
6. **A6** — fix the example after the underlying ergonomics (D1, D2)
   are in.

Re-run `mypy --strict`, `pytest`, and `rig bench` at the end. Update
`CHANGELOG.md`. Write a final paragraph at the bottom of this audit
recording what closed and what slipped.

## Disposition report (closed at 2026-05-15)

**Closed:** A1, A2, A3, A4, A5, A6, B1 (via ExecuteResult.consumed_contract_ids),
B3, C1, C2, C3, D1, D2 (RigConfig), D3, D4 (mid-chain blame with the
"real delegations" refinement), D5 (Example 05), E1, E2, E3, E4, E5.

**Outcome:**
- `mypy --strict` clean across 28 source files (was 106 errors, now 0).
- `pytest` 76 passing (up from 61), no skips.
- Five examples run end-to-end (was four).
- `rig bench` produces the same calibrated scores as before; new code
  paths don't degrade benchmark numbers.

**Slipped to v1 roadmap:**
- B2 *as written* (a Rig-level replay-rejection guard at contract
  issuance) — the property test (E3) only asserts the ULID
  uniqueness invariant. A real replay-rejection guard would compare
  incoming contracts against issued IDs and reject; this needs a
  threat-model decision (memory-bounded vs persistent store) and
  defers to v1.
- KMS-backed signer (still v1).
- Per-trace OTel sampling guidance (still v1).
- Mid-chain blame on *multi-level* failures (current implementation
  promotes one level; a deeper "where in the chain was the bad
  composition?" analysis is v1+).

The audit cycle proved out the discipline that v0 set up: every
fix was attributable to a specific spec section or ADR, no
architectural decisions had to be revisited, and the code shrank in
two places (the cli `type: ignore` lines, the `_open_parent_builder`
stub) while gaining real capability everywhere else.
